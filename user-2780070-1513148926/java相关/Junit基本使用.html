Junit基本使用

用Junit做单元测试非常简单。

1.最近的junit依赖地址，也可以引入环境自带的Junit4的依赖包。

2.通常而言，单元测试路径和源码路径是平级的，下述是一种可以参考的主路径设置方式

src/main/java

src/test/java

src/main/resources

src/test/resources

3.Junit新建类有两种Junit Test Caes和Junit Test suite，前者是单元测试单元后者是一套单元测试集合。

4.Junit创建时会有4个可选初始化类，static setUpBeforeClass,static tearDownAfterClass,setUp,tearDown

前两个为静态的，分别在类第一次使用时和结束使用时执行。后两个为类实例创建时和结束时执行的。Junit对于方法测试是的原理是，每次测试一个独立的方法时都会创建一个新实例。所以如果要做实例初始化，可以在setUp中做。

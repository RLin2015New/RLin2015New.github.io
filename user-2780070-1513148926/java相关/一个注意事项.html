一个注意事项

今天读接口和内部类章节时，发现作者枚举的范例，有一些是明显接触过的，但是只是当作项目代码来理解而没有在概念里形成“解决方案”的概念。

蓦然发现一个学计算机很容易陷入的误区，那就是忽略内容的原意图。

因为计算机算是“舶来品”，所以所学概念和内容，大部分都是源自国外。经过不同的作者不同的翻译以及一些不得已的直译，很多东西其实都有些混乱了。一个概念可能有不同解读也可能被扩散进各种附加的含义。这也是要进阶计算机能力的困难之一。因为体系混乱，如果无法弄清楚真实计算机世界的架构体系，那么就只能自己补全体系。而且，读计算机书，又一个很容易步入的误区就是过度的解读。老是去想作者这句话或概念是什么意思。而实际呢，作者翻译的内容本身可能就是不准确的。

所以，以后学计算机的相关内容。要更注重其英文含义，以及设计意图。而不要只停留在解读书本的层面上。

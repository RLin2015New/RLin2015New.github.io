MemCached的原理非常简单，就是用一个独立进程在内存中占一个区域，这个进程提供对外的访问地址和端口，并约定数据存储的方式。外部程序可以通过第三方包（或自定义包）来访问memcached的进程。这么做有什么好处呢？1.存储的数据不依赖于Jvm，这也就意味着即便宕机数据依然可恢复，而这些数据又不需要做持久化。2.高效的读写速度，以及可控的数据管理。相较于数据库而言，肯定是高速的，因为数据都是直接存储在内存区，而用独立进程约定存储格式，又让存储的数据相较于直接用Jvm存储而言更便于管理。3.可对多个服务器提供服务，用来保证数据一致性。

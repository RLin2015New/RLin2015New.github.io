7.CDN-内容分发网络

  为了提速，希望页面的容量尽量小，浏览器可以尽量多的缓存内容。

  如果不同网站使用相同的JavaScript框架，把框架库存放在一个通用的位置提供给每个网页分享就会变得很有意义。CDN（Content Delivery Network）解决了这个问题。CDN是包含分享代码库的服务器网络。

（CDN貌似是在线视频的关键，google提供了CDN，但是考虑到GFW，还是用国内的CDN比较好）

  CDN表现在html里，就是依赖包的src也就是下载路径。如果依赖的下载和其他网页一样，就有可能用到浏览器缓存而无需再次下载。所以这里，有时候先打开了其他不相关的网页，有可能也会对你的网页打开速度有提升。（情形核心就是其他网页恰好和你的网页有相同的包依赖且使用了一样的CDN）

8.因为通过CDN来下载包，所以在项目起始时，最好对CDN有一个简单测试。确保其基本稳定。

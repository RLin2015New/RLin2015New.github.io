Objective-C之Block

最近在做项目重构，用了比较多的block块。用这个有一个好处就是减少了代码内的函数，让逻辑比较直观，而且可以比较好的定义代码执行的顺序。使用的时候有一个担心就是线程安全问题，也比较好奇block的本质是个什么东东。所以今晚就稍微看了一下。

相关问题可以关注OC中， block（块）的本质是什么？ - Joshua Shen 的回答

block定义

struct Block_descriptor {

unsigned long int reserved;

unsigned long int size;

void (*copy)(void *dst, void *src);

void (*dispose)(void *);

};

struct Block_layout {

void *isa;

int flags;

int reserved;

void (*invoke)(void *, ...);

struct Block_descriptor *descriptor;

/* Imported variables. */

};

定义block之后，实际是创造了一个static函数，在创造block结构体时把函数指针给block持有，所以之后可以调用。

有一个要注意的是，block的参数采取的是值传递，只有在采用__block前缀时才是对象传递。
